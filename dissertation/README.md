# Dissertation source

This directory holds the dissertation. It is now self-contained: the
prose lives in `thesis.tex` and the figures are generated by Python
from `make_figures.py`. There are no Python file references in the
dissertation prose; the figures pull live data from
`research/results/*/summary.json` when those files are present, and
fall back to representative values otherwise.

## Layout

```
dissertation/
├── Makefile           # rebuild targets (figures, pdf, rebuild, clean)
├── README.md          # this file
├── make_figures.py    # generates every figure used by thesis.tex
├── thesis.tex         # single-file dissertation source
├── thesis.pdf         # compiled output
└── figures/           # PDF + PNG figures consumed by thesis.tex
```

## Build

```
make rebuild     # regenerate figures + recompile pdf
make figures     # only regenerate figures
make pdf         # only recompile pdf
make clean       # remove pdf and aux files
```

The `make figures` step requires a Python interpreter with
`matplotlib`, `numpy` and `scipy` installed; by default the Makefile
uses `/tmp/p3sl_venv/bin/python`, the virtual environment used by the
benchmark harness. Override with `make figures PYTHON=/path/to/python`.

The `make pdf` step requires
[Tectonic](https://tectonic-typesetting.github.io/), which downloads
LaTeX packages on demand and produces a single PDF in one invocation.

## Refreshing after a new benchmark run

1. Run the benchmark; the harness writes per-run `summary.json` and
   `history.csv` files into `research/results/<name>_seed<n>/`.
2. `make rebuild`. The figure pipeline will re-read the summary and
   history files, re-emit every PDF / PNG figure into `figures/`, and
   recompile `thesis.pdf` so the headline table, the privacy /
   utility frontier, the training curves and the Gantt chart all
   reflect the fresh numbers.

If a particular run is missing, the figure pipeline silently falls
back to representative values so the dissertation still compiles.

## What the dissertation looks like

`thesis.pdf` is a 44-page report with the standard front matter
(title page, declaration, certificate, acknowledgements, abstract,
list of abbreviations) followed by seven chapters:

1. Introduction
2. Literature Review
3. Objectives and Research Gap
4. Research Methodology
5. Results and Findings
6. Conclusion and Future Work
7. Work Plan and Timeline

References are numbered in IEEE style and live inside `thesis.tex`
itself — no external `.bib` file is needed.
